# Contributing to Trinetra

Thanks for contributing to Trinetra.

## Development Workflow

1. Create a feature branch from `main`.
2. Make focused changes with clear commit messages.
3. Run local checks before opening a PR.
4. Open a pull request using the project PR template.

## Local Checks

```bash
npm run lint
npm run test
npm run build
```

## Coding Standards

- Use TypeScript with strict typing.
- Keep changes aligned to feature boundaries.
- Prefer small, testable functions.
- Add tests for planner/memory/orchestrator logic when behavior changes.

## Pull Request Expectations

- Describe what changed and why.
- Link related issues.
- Note risks and test coverage.
- Keep scope focused; avoid unrelated refactors.

## Issues

Use issue templates for:

- Feature requests
- Bug reports
- Technical debt
